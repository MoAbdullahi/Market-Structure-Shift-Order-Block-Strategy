# CRT + Market Structure Shift + Order Block Strategy

## Implementation Task Report

### Objective

Develop a modular Python backtesting engine implementing the Candle Range Theory (CRT) liquidity reversal strategy using a two-timeframe hierarchy.

The implementation must faithfully reproduce the trading logic while eliminating look-ahead bias, supporting realistic execution modeling, and providing a framework suitable for parameter optimization and future strategy extensions.

---

# Strategy Overview

The strategy combines higher-timeframe liquidity sweeps with lower-timeframe confirmation before entering a retracement into an Order Block.

Timeframes

• Higher Timeframe (HTF): 1 Hour

• Lower Timeframe (LTF): 5 Minutes

Trading workflow

1. Detect HTF CRT trigger.

2. Confirm reversal through LTF Market Structure Shift (MSS).

3. Identify the Order Block responsible for the move.

4. Wait for retracement into the Order Block.

5. Execute limit entry.

6. Manage trade using staged exits and dynamic stop movement.

---

# Functional Requirements

## 1. Higher Timeframe CRT Detection

Every completed H1 candle is evaluated against the previous completed H1 candle.

Bearish CRT

Conditions

• Current High > Previous High

• Current Close < Previous High

• Previous candle range ≥ 0.5 × ATR(14)

Bullish CRT

Conditions

• Current Low < Previous Low

• Current Close > Previous Low

• Previous candle range ≥ 0.5 × ATR(14)

Output

Each valid trigger produces a CRT signal containing

• direction

• trigger timestamp

• trigger high

• trigger low

• previous high

• previous low

---

## 2. Lower Timeframe MSS Confirmation

After a completed HTF trigger is detected, monitor the M5 chart.

The strategy searches for a Market Structure Shift.

Bearish setup

• Determine reference low from the first configurable number of M5 candles.

• Confirm when a candle closes below that reference.

Bullish setup

• Determine reference high.

• Confirm when a candle closes above it.

The MSS confirms that the HTF liquidity sweep has resulted in a genuine directional shift.

No trade is permitted before MSS confirmation.

---

## 3. Order Block Detection

After MSS confirmation

Locate the final opposite-colored candle immediately preceding the impulsive break.

Bearish

Last bullish candle.

Bullish

Last bearish candle.

The candle body defines the Order Block.

The entry price is configurable.

Supported pricing modes

• Candle midpoint

• Open

• Close

• Body high

• Body low

---

## 4. Pending Entry

After identifying the Order Block

Place a pending limit order.

The position opens only if price revisits the Order Block.

No market entries are allowed.

# Required Improvements

## 1. Time Synchronization

Problem

Many implementations immediately begin scanning M5 candles while the H1 candle is still forming.

This introduces look-ahead bias because the final H1 close is unknown until the candle completes.

Required implementation

The engine shall

Wait until the H1 candle closes.

Only then generate the CRT signal.

Begin scanning M5 candles starting with the first completed M5 candle after the H1 close.

Benefits

Eliminates look-ahead bias.

Produces realistic chronological execution.

Matches live trading conditions.

---

## 2. Pending Order Expiration

Problem

Without expiration, a pending Order Block entry remains active indefinitely.

This creates unrealistic fills several hours or days after the market context has changed.

Required implementation

Each pending order shall have a configurable lifetime measured in M5 candles.

Example

PendingLifetime = 24

If the Order Block is not revisited within 24 completed M5 candles

Cancel the order.

Return to Idle.

Benefits

More realistic execution.

Reduces stale entries.

Allows optimization of entry timing.

---
